**AI-Консультант "Пиранья"** — это интеллектуальный Telegram-бот, разработанный для автоматизации продаж и консультирования клиентов типографии "Пиранья". Бот использует архитектуру RAG (Retrieval-Augmented Generation) для предоставления точных ответов на основе внутренней базы знаний о продуктах и услугах, а также ведет клиента по воронке продаж от выбора товара до расчета стоимости.

## ✨ Ключевые возможности

*   **Интеллектуальные консультации (RAG):** Бот отвечает на вопросы клиентов, основываясь на документах из приватной базы знаний (`app/knowledge_base`), используя векторный поиск в **ChromaDB**.
*   **Гибкая воронка продаж (FSM):** Реализована полноценная, многошаговая воронка для заказа продукции (выбор материала, размера, качества печати, обработки, количества, срочности, доставки).
*   **Виджет-калькулятор:** Вместо создания новых сообщений, бот обновляет единый виджет-калькулятор, который всегда отображает актуальный состав и стоимость заказа, не засоряя чат.
*   **Закрепление виджета:** "Виджет-калькулятор" автоматически закрепляется вверху чата, обеспечивая пользователю постоянный доступ к актуальному расчету или последнему заказу.
*   **Гибкий диалог:** Пользователь может на любом этапе воронки задать уточняющий вопрос AI-консультанту, не прерывая процесс заказа.
*   **Память диалогов:** Бот поддерживает контекст диалога в рамках одной сессии, что позволяет задавать уточняющие вопросы.
*   **Отказоустойчивость LLM:** Интеграция с **OpenRouter** и механизм фолбэка обеспечивают стабильную работу: при сбое основной модели система автоматически переключается на резервную.
*   **Уведомление менеджера:** После подтверждения заказа бот автоматически отправляет детали заказа в специальный чат для менеджера.
*   **Комплексный сбор статистики:** Реализован механизм сбора статистики по всем действиям пользователя, включая команды, текстовые запросы и **каждый шаг внутри воронки продаж**, что позволяет проводить детальный анализ конверсии.
*   **Модульная архитектура:** Код четко разделен на слои (`handlers`, `core`, `keyboards`, `utils`), что упрощает поддержку и расширение.
*   **Комплексное тестирование:** Проект имеет 100% покрытие модульными тестами с использованием `pytest`. Тесты охватывают все ключевые компоненты: обработчики `aiogram`, логику FSM и воронки продаж, "Виджет-калькулятор", работу с базами данных и утилиты. Внешние зависимости (API Telegram, LLM) полностью мокируются, что обеспечивает изоляцию и надежность тестов.
*   **Готовность к контейнеризации:** Проект полностью готов к развертыванию с помощью Docker, с автоматической индексацией базы знаний при сборке образа.

## ⚙️ Архитектурные компоненты

### 1. Ядро RAG (Retrieval-Augmented Generation)
Архитектура RAG является основой бота. Она позволяет генерировать ответы, основываясь не на общих знаниях модели, а на конкретных документах из приватной базы знаний.
- **Векторизация:** Для преобразования текста в векторы используется собственный API-сервис (`EMBEDDING_SERVICE_URL`), который работает на базе модели `intfloat/multilingual-e5-large`. Это обеспечивает полный контроль над процессом.
- **Векторное хранилище:** В качестве векторной базы данных используется **ChromaDB**. Она хранит векторы документов локально в директории `chroma_db/` и обеспечивает быстрый семантический поиск релевантного контекста.

### 2. Оркестрация LLM и отказоустойчивость
- **Оркестратор:** **LangChain** используется для построения и управления всей логической цепочкой (RAG-chain).
- **LLM-шлюз:** **OpenRouter** применяется как единая точка доступа к различным большим языковым моделям (LLM).
- **Механизм фолбэка:** Реализована система отказоустойчивости. Если основная модель (`OPENROUTER_CHAT_MODEL`) недоступна или возвращает ошибку, система автоматически переключается на запасную модель (`OPENROUTER_FALLBACK_MODEL`), обеспечивая стабильность работы.

### 3. Воронка продаж и Виджет-калькулятор
- **Конечный автомат (FSM):** Для управления состоянием пользователя внутри воронки продаж используется `FSMContext` из `aiogram`. Класс `OrderState` определяет все возможные шаги (например, `entering_size`, `choosing_print_quality`), что обеспечивает строгую последовательность и логику процесса заказа.
- **Виджет-калькулятор:** Вместо отправки нового сообщения на каждом шаге, бот редактирует одно и то же сообщение-виджет. Логика инкапсулирована в функции `_update_calculator_widget`, которая интеллектуально решает, как обновить сообщение:
  - **`_edit_message`:** "Тяжелая" операция, которая используется при смене типа контента (например, добавление/удаление фото). Она может удалить старое сообщение и отправить новое.
  - **`_update_calculator_message`:** "Легкая" операция, которая просто редактирует текст или подпись к фото, если изображение менять не нужно. Это обеспечивает плавное и эффективное обновление UI.

### 4. Память диалога
- **Хранилище:** Для сохранения истории переписки с каждым пользователем используется база данных **SQLite** (`chat_history.sqlite3`).
- **Механизм:** Класс `SQLiteChatMessageHistory` обеспечивает асинхронный доступ к истории. `ConversationBufferWindowMemory` из LangChain ограничивает контекст последними `MEMORY_WINDOW_SIZE` сообщениями, что оптимизирует расход токенов.

### 5. Калькулятор цен
- **Модуль `pricing.py`:** Вся логика расчета стоимости вынесена в отдельный модуль. Функция `calculate_price` принимает полный набор параметров заказа (материал, размеры, срочность, доставка и т.д.) и возвращает итоговую стоимость на основе прайс-листа, заложенного в коде.

### 6. Пользовательский интерфейс (UI)
- **Фреймворк:** **aiogram 3.x** используется для асинхронного взаимодействия с Telegram API.
- **Интерактивные меню:** Реализованы многоуровневые `inline`-клавиатуры для навигации по воронке продаж.

### 7. Отказоустойчивая отправка ответов и UX
- **Назначение:** Реализован многоуровневый механизм, гарантирующий доставку ответа от AI пользователю даже в случае ошибок форматирования Markdown.
- **Логика:** Вся логика инкапсулирована в обработчике `process_query` и состоит из трех этапов:
  1. **Первая попытка:** Ответ AI проходит "косметическую" обработку (удаление символов `#`) и отправляется с `parse_mode="Markdown"`.
  2. **Аварийное экранирование:** В случае ошибки парсинга, к тексту применяется функция `escape_markdown_legacy`, которая экранирует минимальный набор проблемных символов, и делается вторая попытка отправки.
  3. **Гарантированная доставка:** Если и вторая попытка неудачна, исходный "сырой" ответ AI отправляется как обычный текст (`parse_mode=None`), что обеспечивает 100% доставку сообщения.

- **"Тихое" закрепление:** Для закрепления виджета-калькулятора используется функция `_pin_message_and_delete_notification`, которая сначала закрепляет сообщение, а затем немедленно удаляет системное уведомление об этом. Это обеспечивает чистоту чата и улучшает пользовательский опыт.

### 8. Сбор статистики и Тестирование
- **Сбор статистики:** Модуль `app/core/stats.py` отвечает за логирование всех действий пользователя (команды, нажатия кнопок, текстовые запросы) в таблицу `query_stats` базы данных SQLite. Это позволяет анализировать использование бота и эффективность RAG-системы.
- **Тестирование:** В директории `tests/` находится полный набор модульных тестов, написанных с использованием `pytest`. Тесты покрывают все ключевые компоненты приложения, включая обработчики `aiogram`, логику работы с базами данных и утилиты. Внешние зависимости (API Telegram, LLM) полностью мокируются, что обеспечивает изоляцию и надежность тестов.

## ⛓️ Логика обработки запроса (RAG-цепочка)

Весь процесс ответа на вопрос пользователя инкапсулирован в функции `process_query` и `get_rag_chain`.

1.  **Получение запроса:** Обработчики в `app/handlers/user_handlers.py` принимают текстовое сообщение или `callback_query` от пользователя.
2.  **Вызов цепочки:** Вызывается `process_query`, которая в свою очередь асинхронно запускает `conversational_rag_chain`.
3.  **Загрузка истории:** `_get_async_chat_history` извлекает последние сообщения из SQLite для текущей сессии.
4.  **Поиск контекста:** `retriever` (ChromaDB) находит в базе знаний фрагменты документов, семантически близкие к вопросу пользователя.
5.  **Формирование промпта:** `ChatPromptTemplate` собирает системный промпт, историю чата, вопрос пользователя и найденный контекст в единый запрос для LLM.
6.  **Генерация ответа:** LLM (с механизмом фолбэка) генерирует ответ.
7.  **Отправка и сохранение:** Ответ отправляется пользователю через отказоустойчивый механизм. После этого вопрос пользователя и ответ AI сохраняются в историю чата в SQLite.

## 📦 Инструкции по запуску

### Локальный запуск

1.  **Клонируйте репозиторий и перейдите в директорию**
    ```bash
    git clone https://github.com/mynamemyway/ai-piranha-bot.git
    cd ai-piranha-bot
    ```

2.  **Настройте переменные окружения**
    Создайте файл `.env` в корневой папке проекта.
    ```bash
    touch .env
    ```
    Откройте файл `.env` и добавьте в него следующие переменные, заменив значения на ваши:
    ```dotenv
    # Telegram Bot Token from @BotFather
    BOT_TOKEN="YOUR_TELEGRAM_BOT_TOKEN"

    # Telegram Chat ID for manager notifications
    MANAGER_CHAT_ID="YOUR_MANAGER_CHAT_ID"

    # OpenRouter API Key from https://openrouter.ai/keys
    OPENROUTER_API_KEY="YOUR_OPENROUTER_API_KEY"

    # URL for the self-hosted embedding service API
    EMBEDDING_SERVICE_URL="http://your_server_ip:8000/embed"

    # (Optional) System prompt for the RAG chain.
    SYSTEM_PROMPT="Ты — креативный продавец-консультант..."
    ```

3.  **Запустите сервис эмбеддингов**
    Этот проект требует запущенного e5-large-embedding-service. Убедитесь, что Docker-контейнер с этим сервисом запущен и доступен по адресу, указанному в `EMBEDDING_SERVICE_URL`.

4.  **Наполните базу знаний**
    Поместите ваши документы (в формате `.md`) в директорию `app/knowledge_base/`.

5.  **Запустите бота**
    Перед запуском убедитесь, что все зависимости установлены (`pip install -r requirements.txt`).
    ```bash
    python main.py
    ```

### Запуск с помощью Docker

Проект полностью сконфигурирован для развертывания с помощью Docker Compose, что значительно упрощает процесс.

1.  **Подготовьте окружение**:
    *   Клонируйте репозиторий.
    *   Создайте и настройте файл `.env`.
    *   Убедитесь, что сервис эмбеддингов запущен и доступен.
    *   Наполните директорию `app/knowledge_base` вашими документами.

2.  **Сборка образа и запуск**
    Эта команда соберет Docker-образ. На этапе сборки автоматически запустится скрипт индексации (`app.core.rag`), который создаст векторную базу знаний и включит ее в образ. После сборки сервис запустится в фоновом режиме.
    ```bash
    docker-compose up --build -d
    ```
    *   **Автоматическая индексация:** Векторный индекс создается при сборке и является частью образа.
    *   **Перезапуск:** Контейнер будет автоматически перезапускаться в случае сбоев благодаря политике `restart: unless-stopped`.
    *   **Сохранение данных:** Все необходимые тома для `ChromaDB` и `SQLite` настроены в `docker-compose.yml` для сохранения данных между перезапусками.
    *   **Просмотр логов:** `docker-compose logs -f`
    *   **Остановка сервиса:** `docker-compose down`
