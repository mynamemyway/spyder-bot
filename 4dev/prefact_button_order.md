### План реализации отправки заказа менеджеру

Предварительное требование:
7. Не все кнопки попадают в query_stats , например ввод размера баннера - расчёты не сохраняются.
8. Добавить хранение отправленных менеджеру заказов?
Либо можно создать отдельную таблицу, в которую будут попадать сделанные расчёты и добавить флаг is_order, который будет показывать, отправлен ли расчёт менеджеру кнопкой "Заказать"?

1.  **Модификация `app/config.py`:**
    *   **Цель:** Добавить ID чата менеджера для отправки уведомлений.
    *   **Действие:** В класс `Settings` добавить новый обязательный атрибут `MANAGER_CHAT_ID: int`.

2.  **Модификация `app/keyboards.py`:**
    *   **Цель:** Создать клавиатуру с `callback`-кнопкой для инициирования отправки заказа.
    *   **Действие:** Создать функцию `get_order_keyboard()`, которая генерирует `InlineKeyboardMarkup` с кнопкой "✅ Заказать", использующей `ActionCallback(name="create_order")`.

3.  **Модификация `app/handlers/user_handlers.py`:**
    *   **Цель:** Реализовать полную логику сохранения данных о заказе и отправки уведомления менеджеру.
    *   **Действие 1 (в `handle_size_input`):**
        *   После расчета цены, сохранить `width`, `height` и `price` в FSM-состоянии.
        *   Не очищать состояние после расчета.
        *   Прикрепить к сообщению с ценой клавиатуру `get_order_keyboard()`.
    *   **Действие 2 (создание нового обработчика):**
        *   Создать новый обработчик, срабатывающий на `ActionCallback(name="create_order")`.
        *   Внутри него: извлечь данные из FSM, отправить уведомление на `MANAGER_CHAT_ID`, подтвердить отправку пользователю, убрать кнопку "Заказать" и очистить состояние.


### План реализации отправки заказа менеджеру с сохранением в БД 2.0

1.  **Модификация `app/config.py`:**
    *   **Цель:** Добавить ID чата менеджера для отправки уведомлений.
    *   **Действие:** В класс `Settings` добавить новый обязательный атрибут `MANAGER_CHAT_ID: int`.

2.  **Создание модуля `app/core/orders.py` (Новый файл):**
    *   **Цель:** Инкапсулировать логику взаимодействия с таблицей `orders`.
    *   **Действие 1:** Создать функцию `_create_orders_table()` для инициализации таблицы `orders` в SQLite.
    *   **Действие 2:** Создать асинхронную функцию `log_order(...)`, которая сохраняет детали заказа в таблицу и возвращает `id` новой записи.

3.  **Модификация `app/keyboards.py`:**
    *   **Цель:** Создать клавиатуру с `callback`-кнопкой для инициирования отправки заказа.
    *   **Действие:** Создать функцию `get_order_keyboard()`, которая генерирует `InlineKeyboardMarkup` с кнопкой "✅ Заказать", использующей `ActionCallback(name="create_order")`.

4.  **Модификация `app/handlers/user_handlers.py`:**
    *   **Цель:** Реализовать полную бизнес-логику сохранения и отправки заказа.
    *   **Действие 1 (в `handle_size_input`):**
        *   Сохранить данные о расчете (`width`, `height`, `price`) в FSM-состоянии.
        *   Прикрепить к сообщению с ценой клавиатуру `get_order_keyboard()`.
    *   **Действие 2 (создание нового обработчика `handle_create_order`):**
        *   Создать обработчик для `ActionCallback(name="create_order")`.
        *   **Логика:** Извлечь данные из FSM -> вызвать `log_order()` для сохранения в БД -> отправить уведомление менеджеру с `order_id` -> отправить подтверждение пользователю с `order_id` -> убрать кнопку -> очистить состояние.
