На основе анализа я подготовил пошаговый план рефакторинга. Он направлен на сохранение стабильного ядра RAG и пошаговую адаптацию UI и бизнес-логики под новую задачу.

- **Обновление Dockerfile:** Добавить шаг для запуска скрипта индексации `app.core.rag` непосредственно во время сборки Docker-образа. Это гарантирует, что каждый новый образ будет содержать актуальную, готовую к работе векторную базу.
- **Обновление README.md:** Адаптировать документацию, отразив новую цель проекта и изменив инструкции по запуску и обновлению базы знаний в соответствии с новой логикой сборки.

## Этап 3: Адаптация Системного Промпта и Логики RAG

**Задача:** Изменить "личность" и поведение AI, чтобы он соответствовал роли продавца-консультанта.

## Этап 4: Реализация Калькулятора Цен (MVP)

**Шаги:**

- **Интеграция с обработчиками:** В `user_handlers.py` добавить логику, которая по мере выбора пользователем опций (продукт, материал, размер) собирает информацию о заказе и в конце воронки передает ее в модуль `pricing.py` для расчета и вывода итоговой цены.

**Цель:** Сохранение стабильного ядра RAG и пошаговая адаптация UI и бизнес-логики под новую задачу. Рефакторинг существующего Telegram-бота "Portfolio AI" для создания AI-консультанта по продажам для типографии "Пиранья". Проект будет сфокусирован на реализации структурированной воронки продаж, начиная с MVP для продукта "Баннеры".

## Этап 1: Адаптация Архитектуры и Конфигурации
**Задача:** Подготовить проект к работе с новыми данными и реализовать архитектуру, при которой база знаний и ее векторный индекс являются неотъемлемой частью Docker-образа.

### 1.2. Интеграция Индексации в Сборку Образа:

## Этап 2: Рефакторинг Пользовательского Интерфейса (UI)
**Задача:** Заменить UI, ориентированный на портфолио, на UI, реализующий воронку продаж для выбора баннеров.

### 2.1. Переработка Клавиатур (keyboards.py):

- Создать новые inline-клавиатуры для каждого шага воронки: выбор продукта (начнем с "Баннеры"), выбор материала (глянец/матовый), выбор размера и т.д.

### 2.2. Переработка Обработчиков (user_handlers.py):

- Адаптировать существующие callback query handlers для управления навигацией по новой многоуровневой клавиатуре. Логика должна последовательно вести пользователя по шагам воронки.

### 2.3. Обновление Команд Меню (ui_commands.py):

- Заменить команды меню Telegram (`/start`, `/help`, `/reset`) на более релевантные для бота-консультанта (например, `/start` - начало диалога, `/catalog` - переход к выбору продукции, `/help` - помощь).

## Этап 3: Адаптация Системного Промпта и Логики RAG
**Задача:** Изменить "личность" и поведение AI, чтобы он соответствовал роли продавца-консультанта.

### 3.1. Обновление Системного Промпта:
- Изменить значение переменной `SYSTEM_PROMPT` в файле `.env`. Новый промпт должен четко инструктировать LLM о его роли (продавец-консультант типографии "Пиранья"), задачах (вести по воронке, консультировать, рассчитывать цену) и стиле общения (клиентоориентированный, экспертный).

## Этап 4: Реализация Калькулятора Цен (MVP)
**Задача:** Добавить базовую логику расчета стоимости заказа на основе выбора пользователя.

### 4.1. Создание Модуля Расчета Цен (app/core/pricing.py):

- Реализовать простой механизм хранения цен, например, в виде словаря Python или .json файла, где ключами будут являться параметры заказа (продукт, материал, размер).
- Создать функцию, которая принимает параметры заказа и возвращает итоговую стоимость.

### 4.2. Интеграция Калькулятора в Воронку:

- В `user_handlers.py` добавить логику, которая по мере выбора пользователем опций сохраняет его выбор (например, в FSMContext от aiogram).
- На последнем шаге воронки вызывать функцию из `pricing.py` с собранными параметрами и выводить пользователю рассчитанную цену.
