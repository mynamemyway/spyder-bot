### **Часть 0: Клонирование репозитория в папку проектов**

*Выполняется один раз при первоначальной настройке.*

1.  **Проверь, какие папки есть в домашней директории:**
    ```bash
    ls -la /home/mynamemyway/
    ```

2.  **Удали репозиторий из домашней директории (если был склонирован туда):**
    ```bash
    rm -rf ~/ai-spyder-bot
    ```

3.  **Перейди в директорию проектов и клонируй репозиторий туда:**
    ```bash
    cd /home/mynamemyway/projects
    git clone git@github.com:mynamemyway/ai-spyder-bot.git
    ```

> **Важно:** Репозиторий должен находиться в `/home/mynamemyway/projects/ai-spyder-bot`

### **Часть 1: Обновление файла `.env` на сервере**

*Эта операция выполняется на твоём локальном компьютере (MacBook).*

1.  **Открой терминал на локальной машине.**

2.  **Выполни команду для безопасного копирования файла `.env` на сервер.**
    Эта команда использует `scp` (secure copy) для передачи файла по SSH. Она заменит старый файл `.env` на сервере новым с твоей локальной машины.

    ```bash
    scp ~/Documents/GitHub/ai-spyder-bot/.env mynamemyway@91.206.14.93:/home/mynamemyway/projects/ai-spyder-bot/.env
    ```
    *   `~/Documents/GitHub/ai-spyder-bot/.env` — это путь к твоему локальному файлу.
    *   `mynamemyway@91.206.14.93:/home/mynamemyway/projects/ai-spyder-bot/.env` — это путь назначения на сервере.

### **Часть 2: Деплой нового релиза**

*Эти операции выполняются сначала на локальной машине, а затем на сервере.*

1.  **На локальной машине: убедись, что все изменения загружены в репозиторий.**
    *   Открой GitHub Desktop.
    *   Сделай коммит всех последних изменений.
    *   Нажми "Push origin", чтобы отправить изменения на GitHub.

2.  **На сервере: подключись и обнови приложение.**
    *   **Подключись к серверу по SSH:**
        ```bash
        ssh mynamemyway@91.206.14.93
        ```

    *   **Перейди в директорию проекта:**
        ```bash
        cd /home/mynamemyway/projects/ai-spyder-bot
        ```

    *   **Останови контейнеры:**
        *Останавливаем контейнер бота. Тома данных сохраняются.*
        ```bash
        docker-compose down
        ```

    *   **Удали том с базой знаний ChromaDB:**
        *Это нужно для пересоздания индекса с актуальными данными. Том `sqlite_data` с историей чатов сохраняется.*
        ```bash
        docker volume rm ai-spyder-bot_chroma_data
        ```
        > **Примечание:** Если команда возвращает ошибку "volume not found", значит том ещё не был создан — это нормально для первого деплоя.

    *   **Загрузи последние изменения из репозитория:**
        ```bash
        git pull
        ```

    *   **Пересобери Docker-образ и перезапусти сервис:**
        Эта команда остановит старые контейнеры, пересоберет образ с учётом новых файлов (включая автоматическую индексацию базы знаний) и запустит новые контейнеры в фоновом режиме.
        ```bash
        docker-compose up --build -d
        ```

    *   **(Опционально) Очисти старые образы:**
        Чтобы освободить место на диске, можно удалить старые, неиспользуемые Docker-образы.
        ```bash
        docker image prune -f
        ```

    *   **Проверь логи, чтобы убедиться, что всё запустилось корректно:**
        ```bash
        docker-compose logs -f
        ```
        Нажми `Ctrl + C`, чтобы выйти из режима просмотра логов.

### **Часть 3: Проверка работы**

1.  **Проверь статус контейнера:**
    ```bash
    docker-compose ps
    ```
    Статус должен быть `Up`.

2.  **Протестируй бота в Telegram:**
    *   Отправь сообщение боту в личном чате.
    *   Упомяни бота в группом чате: `@spyder_troitskiy_bot, привет`.

3.  **Проверь логи на ошибки:**
    ```bash
    docker-compose logs bot | tail -50
    ```

### **Часть 4: Быстрый перезапуск после обновления `.env`**

*Если ты обновил только файл `.env` (например, изменил `SYSTEM_PROMPT` или `REPLY_ONLY_TO_MENTIONS`), **не нужно пересобирать образ**.*

**На сервере по SSH:**

0.  **Подключись к серверу:**

    ```bash
    scp ~/Documents/GitHub/ai-spyder-bot/.env mynamemyway@91.206.14.93:/home/mynamemyway/projects/ai-spyder-bot/.env

    ```

1.  **Подключись к серверу:**

    ```bash
    ssh mynamemyway@91.206.14.93
    ```

2.  **Перейди в директорию проекта:**
    ```bash
    cd /home/mynamemyway/projects/ai-spyder-bot
    ```

3.  **Перезапусти контейнер (новый `.env` применится автоматически):**
    ```bash
    docker-compose restart
    ```

    *Или останови и запусти заново:*
    ```bash
    docker-compose down
    docker-compose up -d
    ```

> **Важно:**
> - **Не используй** `--build` — это запустит пересборку образа
> - **Не удаляй** том `chroma_data` — база знаний сохранится
> - Docker Compose автоматически читает актуальный `.env` с хоста при запуске

**Проверка:**
```bash
# Посмотри логи
docker-compose logs -f

# Проверь статус
docker-compose ps
```
