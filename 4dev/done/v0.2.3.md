# Release v0.2.3: UX Improvements and FSM Logic Fix

## Краткое описание

Данный релиз сфокусирован на улучшении пользовательского опыта (UX) и исправлении критического бага в логике навигации по воронке продаж. Были добавлены новые, более интуитивные разделители для ввода размеров, а также скорректирована работа с конечным автоматом (FSM) для обеспечения консистентного поведения бота при возврате на предыдущие шаги.

## Основные изменения

### 1. Расширение разделителей для ввода размера

Для повышения гибкости ввода размеров баннера были добавлены новые разделители. Теперь, помимо `x` и `х`, пользователь может использовать символ `*` и слово `на` (в любом регистре).

-   **Файл:** `app/handlers/user_handlers.py`
-   **Логика:** Регулярное выражение в декораторе `@router.message` и в теле обработчика `handle_size_input` было модифицировано. Центральная часть выражения `[xх]` была заменена на не-захватывающую группу `(?:[xх]|\*|на)`.
-   **Тесты:** В `tests/test_handlers.py` добавлены новые кейсы для проверки работы с разделителями `*` и `на`, обеспечивая полное покрытие нового функционала.

### 2. Исправление логики сброса состояния FSM при навигации назад

Была обнаружена и исправлена ошибка, при которой состояние FSM не сбрасывалось корректно при возврате на общие шаги меню (выбор категории, выбор материала), что приводило к некорректной обработке текстовых сообщений.

-   **Проблема:** Использование `await state.clear()` полностью очищало все данные пользователя в FSM, включая `calculator_message_id`, что приводило к ошибкам `Calculator message ID not found` при попытке обновить виджет.
-   **Решение:** В функции `navigate_catalog` для всех уровней, представляющих собой общие меню (`level=0`, `level=1`, `level=2`), вызов `await state.clear()` был заменен на `await state.set_state(None)`.
-   **Объяснение:** В отличие от `clear()`, метод `set_state(None)` сбрасывает только *состояние* конечного автомата, но **сохраняет все данные** (`data`), включая `calculator_message_id`.

### 3. Обновление документации

-   **`README.md`**: Версия проекта обновлена до `0.2.3`.
-   **`4dev/update_input_size.md`**: Создан документ, подробно описывающий внесенные изменения.
-   **Обновлены примеры для пользователя:** В `app/handlers/user_handlers.py` обновлен текст `MATERIAL_SELECTED_TEXT` для демонстрации новых, более гибких примеров ввода размера.

## Результат

1.  **Улучшенный UX:** Пользователи могут вводить размеры более естественным и интуитивным способом.
2.  **Стабильность воронки:** Устранены ошибки, связанные с потерей ID виджета при навигации назад.
3.  **Консистентное поведение:** Теперь при возврате на общие шаги меню бот корректно обрабатывает текстовые сообщения как запросы к AI, а не как попытку ввода размера, что соответствует ожиданиям пользователя.
