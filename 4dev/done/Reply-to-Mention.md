# План: Режим ответа на упоминания (Reply-to-Mention)

## Задача
Бот должен отвечать только когда его упоминают по нику (например, `@spyder_troitskiy_bot`). В личных сообщениях — отвечает всегда. Добавить переключатель в конфиге.

---

## Этап 1: Конфигурация

### 1.1. Добавить переменную в `app/config.py`
**Файл:** `app/config.py`

**Логика:**
- Добавить поле `REPLY_ONLY_TO_MENTIONS: bool = False`
- По умолчанию `False` — бот отвечает на все сообщения (текущее поведение)
- Если `True` — бот отвечает только на упоминания в чатах

---

## Этап 2: Определение типа чата и упоминания

### 2.1. Обновить обработчик сообщений в `app/handlers/user_handlers.py`
**Файл:** `app/handlers/user_handlers.py`

**Функция:** `handle_message(message: Message, bot: Bot)`

**Логика:**
1. Проверить тип чата:
   - `message.chat.type == "private"` — личный чат, обрабатывать всегда
   - `message.chat.type in ("group", "supergroup", "channel")` — групповой чат, проверять упоминание

2. Для групповых чатов проверить упоминание бота:
   - `bot.mentioned_in_text(message.text)` — встроенный метод aiogram
   - Или проверить наличие `@{bot.username}` в тексте сообщения

3. Если упоминания нет и режим `REPLY_ONLY_TO_MENTIONS = True` — игнорировать сообщение

---

## Этап 3: Тестирование

### 3.1. Проверка сценариев
**Файл:** `tests/test_mention_mode.py` (опционально)

**Сценарии:**
1. Личный чат, `REPLY_ONLY_TO_MENTIONS = False` → бот отвечает
2. Личный чат, `REPLY_ONLY_TO_MENTIONS = True` → бот отвечает
3. Групповой чат, `REPLY_ONLY_TO_MENTIONS = False` → бот отвечает на все сообщения
4. Групповой чат, `REPLY_ONLY_TO_MENTIONS = True`, есть упоминание → бот отвечает
5. Групповой чат, `REPLY_ONLY_TO_MENTIONS = True`, нет упоминания → бот игнорирует

---

## Пример использования

### .env
```dotenv
# Режим ответа: True — только на упоминания, False — на все сообщения
REPLY_ONLY_TO_MENTIONS=True
```

### Поведение
- **Групповой чат:**
  - `«Как дела?»` → бот молчит
  - `«@spyder_troitskiy_bot, как дела?»` → бот отвечает
- **Личный чат:**
  - `«Как дела?»` → бот отвечает всегда

---

## Примечания
- Метод `bot.mentioned_in_text()` доступен в aiogram 3.x
- Альтернатива: ручная проверка через `if f"@{bot.username}" in message.text`
- Важно: `bot.username` может быть `None` до первого вызова API — нужно обработать

# План: Ответ на реплаи (Reply-to-Message)

## Задача
Бот должен отвечать не только на упоминания `@bot_username`, но и когда пользователь отвечает (реплаит) на сообщение бота.

---

## Этап 1: Логика проверки реплая

### 1.1. Обновить функцию `_is_bot_mentioned()` в `app/handlers/user_handlers.py`

**Файл:** `app/handlers/user_handlers.py`

**Логика:**
1. Проверить наличие упоминания `@bot_username` в тексте (существующая логика)
2. **Добавить:** Проверить, является ли сообщение ответом на сообщение бота:
   - `message.reply_to_message` — содержит исходное сообщение
   - `message.reply_to_message.from_user.id == bot.id` — бот ли это

**Сигнатура:**
```python
async def _is_bot_mentioned(message: Message, bot: Bot) -> bool:
    # Проверка упоминания @bot_username
    # + Проверка reply_to_message от бота
```

---

## Этап 2: Тестирование сценариев

### 2.1. Проверка работы

**Сценарии:**
1. Групповой чат, `REPLY_ONLY_TO_MENTIONS=True`:
   - Сообщение без упоминания и без реплая → бот молчит
   - Сообщение с `@bot_username` → бот отвечает
   - Реплай на сообщение бота → бот отвечает ✅
   - Реплай на сообщение другого пользователя → бот молчит

2. Личный чат:
   - Любое сообщение → бот отвечает всегда

## Пример использования

### Структура сообщения в Telegram

```
User: @spyder_troitskiy_bot, как дела?
Bot:  Приветствую, например! Всё моднейше, например!

User: [ответ на сообщение бота] А что нового?
Bot:  [отвечает] Всё в режиме дичайшего угара, например!
```

### Код проверки

```python
# Проверка: ответил ли пользователь на сообщение бота
if message.reply_to_message:
    if message.reply_to_message.from_user.id == bot.id:
        return True  # Бот должен ответить
```

---

## Примечания

1. **`bot.id`** — числовой ID бота (например, `8405798658`)
2. **`bot.get_me()`** — можно использовать для получения ID, если неизвестен
3. **Кэширование:** ID бота можно получить один раз при старте и кэшировать
4. **Группы:** В групповых чатах `reply_to_message` сохраняет автора исходного сообщения

---

## Реализация

Добавить в `_is_bot_mentioned()`:

```python
async def _is_bot_mentioned(message: Message, bot: Bot) -> bool:
    if not message.text:
        return False

    # Проверка реплая на сообщение бота
    if message.reply_to_message:
        bot_me = await bot.get_me()
        if message.reply_to_message.from_user.id == bot_me.id:
            return True

    # Проверка упоминания @bot_username
    bot_username = bot_me.username
    if not bot_username:
        return True

    mention = f"@{bot_username}"
    return mention.lower() in message.text.lower()
```
