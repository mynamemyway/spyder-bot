# Release v0.2.0: "Live Calculator" & Flexible Sales Funnel

Этот релиз знаменует собой завершение второго крупного этапа рефакторинга, направленного на кардинальное улучшение пользовательского опыта и расширение функциональности воронки продаж. Ключевые нововведения — реализация концепции "живого калькулятора" и создание гибкой, многошаговой воронки с возможностью консультации с AI на любом этапе.

---

## I. Реализация "Живого калькулятора"

**Проблема:** Ранее каждый шаг воронки создавал новое сообщение, что засоряло чат и мешало пользователю видеть актуальный расчет.

**Решение:** Внедрена концепция "живого калькулятора" — единого, постоянно обновляемого сообщения-виджета, которое служит центральным элементом взаимодействия в воронке продаж.

-   **Централизованное управление виджетом:**
    -   При старте воронки (`/catalog` или `/start`) ID сообщения-калькулятора сохраняется в FSM-состоянии (`calculator_message_id`).
    -   Создана интеллектуальная функция-обертка `_update_calculator_widget`, которая инкапсулирует всю логику обновления UI. Она автоматически определяет, нужно ли менять изображение (используя "тяжелую" операцию `_edit_message`) или достаточно обновить только текст (используя "легкую" `_update_calculator_message`).
    -   Все обработчики воронки теперь вызывают `_update_calculator_widget`, передавая желаемое состояние, что делает код чистым и декларативным.

-   **Управление жизненным циклом виджета:**
    -   Реализовано автоматическое удаление старого виджета при создании нового (команды `/start`, `/catalog`), что предотвращает накопление неактуальных сообщений.

-   **Визуальное сопровождение:**
    -   Добавлены и интегрированы кастомизированные изображения для ключевых этапов воронки: выбор продукта, расчет и подтверждение заказа (`BANNERS_PHOTO_PATH`, `CALCULATION_PHOTO_PATH`, `ORDER_PHOTO_PATH`).
    -   Исправлен баг, при котором изображения "залипали" при навигации назад. Теперь `_update_calculator_widget` корректно отслеживает и меняет изображение на каждом шаге.

## II. Расширение и гибкость воронки продаж (FSM)

**Проблема:** Воронка была короткой и негибкой, не позволяя пользователю отвлекаться на уточняющие вопросы.

**Решение:** Воронка была значительно расширена и сделана интерактивной.

-   **Новая последовательность шагов:** Реализована полная и логичная последовательность выбора параметров для заказа баннера:
    1.  Выбор материала.
    2.  Ввод размера.
    3.  Выбор качества печати (DPI).
    4.  Выбор типа постобработки.
    5.  Выбор количества (включая ручной ввод).
    6.  Выбор срочности заказа.
    7.  Выбор способа получения (доставка/самовывоз).
    8.  Финальный расчет и подтверждение заказа.

-   **Гибкий диалог:**
    -   Реализован обработчик `handle_question_in_funnel`, который позволяет пользователю на любом этапе воронки задать вопрос AI в свободной форме.
    -   Обработчики ввода данных (например, размера) теперь срабатывают только на конкретный формат (`regexp`), передавая все остальные текстовые сообщения на обработку AI-консультанту. Это позволяет пользователю уточнять детали, не выходя из процесса заказа.

-   **Обновленный калькулятор цен:**
    -   Модуль `app/core/pricing.py` был полностью переработан.
    -   Создана новая структура прайс-листа на основе `calculator_from_website_banners.md`.
    -   Функция `calculate_price` теперь учитывает все новые параметры: `dpi`, `quantity`, `is_urgent`, `needs_delivery`.

## III. Уведомление менеджера и завершение заказа

**Проблема:** Процесс заказа обрывался на ссылке на менеджера без фиксации самого заказа.

**Решение:** Реализован полноценный механизм подтверждения заказа.

-   **Интерактивное подтверждение:** Кнопка "Заказать" теперь является `callback`-кнопкой, запускающей финальный этап.
-   **Уведомление менеджера:** После подтверждения бот автоматически формирует и отправляет в чат менеджеру (`MANAGER_CHAT_ID`) сообщение со всеми деталями заказа.
-   **Обратная связь для пользователя:** Сообщение-калькулятор обновляется в последний раз, показывая статус "Заказ принят!" и предоставляя кнопку для прямой связи с менеджером.

## IV. Исправление ошибок и улучшение стабильности

-   **Ошибки в меню `/help`:** Устранены критические ошибки `NameError` и `Calculator message ID not found`, возникавшие при использовании кнопок "Перезапустить" и "Главное меню". Логика кнопок была исправлена для корректной инициализации нового виджета.
-   **Очистка состояния FSM:** Исправлена ошибка, приводившая к преждевременной очистке FSM-состояния и падению бота при навигации по каталогу.

В результате этих изменений бот получил мощную, гибкую и интуитивно понятную воронку продаж, которая значительно улучшает пользовательский опыт и автоматизирует процесс сбора информации для заказа.
