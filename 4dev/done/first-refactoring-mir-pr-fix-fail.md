### План реализации отказоустойчивого санитайзера (Версия 2.1)

1.  **Модификация `app/utils/text_formatters.py`:**
    *   **Цель:** Создать две независимые функции для косметической и аварийной обработки текста.
    *   **Действие 1:** Создать функцию `escape_markdown_legacy(text: str) -> str`. Она будет экранировать обратным слэшем только минимальный набор символов, вызывающих ошибки в `parse_mode="Markdown"`: `_`, `*`, `` ` ``, `[`.
    *   **Действие 2:** Упростить существующую функцию `sanitize_for_telegram_markdown(text: str) -> str` так, чтобы она выполняла только "косметическую" обработку: удаление символов `#` в начале строк.

2.  **Модификация `app/core/chain.py`:**
    *   **Цель:** Убрать из RAG-цепочки любую ответственность за форматирование ответа для Telegram.
    *   **Действие:** Полностью удалить вызов санитайзера из `get_rag_chain`. Цепочка должна возвращать "сырой" текст ответа от LLM.

3.  **Модификация `app/handlers/user_handlers.py`:**
    *   **Цель:** Реализовать трехступенчатый, отказоустойчивый механизм отправки ответа в функции `process_query`, который работает независимо от настройки `SANITIZE_RESPONSE`.
    *   **Действие:**
        1.  Получить "сырой" ответ `ai_response` от RAG-цепочки.
        2.  Определить текст для первой попытки отправки:
            *   Если `settings.SANITIZE_RESPONSE` включен, `text_to_send = sanitize_for_telegram_markdown(ai_response)`.
            *   Иначе, `text_to_send = ai_response`.
        3.  **Первая попытка (try):** Отправить `text_to_send` с `parse_mode="Markdown"`.
        4.  **Обработка ошибки (except TelegramBadRequest):**
            *   Если ошибка связана с парсингом, применить "аварийное" экранирование к **тексту, который не удалось отправить**: `escaped_text = escape_markdown_legacy(text_to_send)`.
            *   **Вторая попытка:** Отправить `escaped_text` с `parse_mode="Markdown"`.
            *   **Третья попытка (вложенный try-except):** Если и вторая попытка вызывает ошибку, перехватить ее и отправить **исходный сырой ответ** `ai_response` с `parse_mode=None` для 100% гарантии доставки.
