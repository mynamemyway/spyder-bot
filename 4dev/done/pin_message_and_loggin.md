# План доработки: Закрепление "Виджета калькулятор" (v2)

**Цель:** Реализовать закрепление "живого калькулятора" при его создании, чтобы он всегда был доступен пользователю вверху чата.

---

### **Этап 1: Модификация `app/handlers/user_handlers.py`**

*   **Задача:** Интегрировать вызов `pin_chat_message` в обработчики, создающие виджет.
*   **Файл:** `app/handlers/user_handlers.py`
*   **Логика:**

    1.  **Закрепление при создании виджета (`handle_start`, `handle_catalog`):**
        *   Сразу после создания нового сообщения-виджета (`widget_message = await ...`) добавить вызов `await bot.pin_chat_message(...)`.
        *   Передать в него `chat_id` и `message_id` нового виджета.
        *   Установить `disable_notification=True`, чтобы избежать лишних уведомлений.
        *   Обернуть вызов в `try...except TelegramBadRequest` для обработки возможных ошибок (например, отсутствие прав на закрепление).

    2.  **Открепление:**
        *   **Не реализуется.** В соответствии с новой логикой, виджет (активный или завершенный) всегда остается закрепленным как точка быстрого доступа для пользователя. Telegram API автоматически заменит старый закреп на новый при вызове `pin_chat_message`.

## Краткое описание

Этот pull request вносит два ключевых улучшения в пользовательский опыт и аналитику:
1.  **Закрепление "живого калькулятора":** Теперь виджет-калькулятор автоматически закрепляется вверху чата при его создании, обеспечивая пользователю постоянный и быстрый доступ к нему.
2.  **Логирование действий в воронке:** Реализован сбор статистики по всем нажатиям кнопок внутри воронки продаж, что позволяет проводить полный анализ поведения пользователей.

---

## I. Закрепление виджета-калькулятора

**Проблема:** После нескольких сообщений в чате "живой калькулятор" уходил вверх по истории, и пользователю приходилось его искать.

**Решение:** Реализован механизм автоматического закрепления виджета.

-   **Надежное "тихое" закрепление:**
    -   Создана новая функция-помощник `_pin_message_and_delete_notification`.
    -   Эта функция сначала закрепляет сообщение с флагом `disable_notification=True`, а затем **немедленно удаляет системное уведомление** о закреплении (`message_id + 1`). Этот подход гарантированно избавляет чат от визуального шума, даже если клиент Telegram игнорирует флаг `disable_notification`.
-   **Интеграция:** Вызов новой функции добавлен в обработчики `handle_start` и `handle_catalog` сразу после создания виджета.
-   **Сохранение истории заказов:** В соответствии с финальным решением, логика открепления сообщений **не реализуется**. Это позволяет пользователю иметь список закрепленных сообщений, который служит удобной историей его активных и завершенных расчетов/заказов.

## II. Логирование действий в воронке продаж

**Проблема:** Ранее действия пользователя внутри многошаговой воронки продаж (выбор материала, качества печати и т.д.) не фиксировались в базе данных статистики, что делало невозможным анализ конверсии.

**Решение:** Добавлено логирование для всех `callback`-запросов в воронке.

-   **Интеграция в `navigate_catalog`:** В начало обработчика `navigate_catalog`, который является единой точкой входа для всей навигации по воронке, добавлен вызов функции `log_query`.
-   **Информативный формат лога:** Действия пользователя записываются в таблицу `query_stats` в формате `CLICK: {action} (item: {item_id})` (например, `CLICK: select_material (item: frontlit_440)`). Это обеспечивает детальный сбор данных о выборе пользователя на каждом шаге.

Эти изменения делают взаимодействие с ботом более удобным для пользователя и предоставляют ценные данные для анализа эффективности воронки продаж.
