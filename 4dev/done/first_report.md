# refactor: Initial Refactoring to AI Sales Consultant MVP

## Краткое описание

Этот pull request завершает первый этап масштабного рефакторинга, трансформируя исходный (личный) проект "Portfolio AI" в MVP (Minimum Viable Product) "AI-Консультант Пиранья". В ходе работы была восстановлена и стабилизирована базовая функциональность, а также реализована новая бизнес-логика, ориентированная на воронку продаж для типографии.

## I. Восстановление и стабилизация базового функционала

После первоначального рефакторинга был выявлен и исправлен ряд критических ошибок, что позволило вернуть боту полную работоспособность.

### 1.1. Восстановление ядра RAG и хранения данных
- **Запросы к AI:** Исправлена ошибка, из-за которой запросы не доходили до LLM. Восстановлена корректная работа `rag_chain.ainvoke`.
- **История чата:** Восстановлен механизм сохранения истории диалогов в `SQLite`. Теперь бот корректно поддерживает контекст беседы с каждым пользователем.
- **Статистика:** Восстановлена работа модуля `app/core/stats.py`. Все действия пользователя (команды, нажатия кнопок, текстовые запросы) снова логируются в базу данных.

### 1.2. Исправление UI и обработки команд
- **Команда `/help`:** Восстановлены `inline`-кнопки "Перезапустить" и "Очистить историю", а также реализован отдельный обработчик `handle_actions` для них.
- **Команда `/reset`:** Восстановлена логика очистки истории чата для текущего пользователя.
- **Изображения для `/start` и `/help`:** Восстановлена логика отправки изображений для команд `/start` и `/help` на основе путей, указанных в `WELCOME_PHOTO_PATH` и `HELP_PHOTO_PATH`.

### 1.3. Реализация отказоустойчивой обработки сообщений
- **Редактирование сообщений:** Полностью переработана логика редактирования сообщений в `_edit_message`. Новая функция корректно обрабатывает все сценарии смены контента (`Фото -> Текст`, `Текст -> Фото`, `Фото -> Другое фото`), используя `edit_media`, `delete`/`send` или `edit_text` по необходимости. Это исправило баг, при котором изображения не менялись при навигации по каталогу.
- **Санитайзер ответов AI:** Реализован трехступенчатый, отказоустойчивый механизм отправки ответов от AI:
  1. **Первая попытка:** Отправка отформатированного текста с `parse_mode="Markdown"`.
  2. **Аварийное экранирование:** В случае ошибки парсинга, применяется `escape_markdown_legacy` для экранирования проблемных символов и делается вторая попытка.
  3. **Гарантированная доставка:** Если и вторая попытка неудачна, исходный "сырой" ответ отправляется с `parse_mode=None`.
- **Форматирование заголовков:** Санитайзер теперь не только удаляет символы `#` из заголовков, но и выделяет их жирным шрифтом, добавляя после них пустую строку для улучшения читаемости.

## II. Новые функции и рефакторинг для воронки продаж

На базе стабильного ядра была реализована новая бизнес-логика, превращающая бота в консультанта по продажам.

### 2.1. Реализация воронки продаж (FSM)
- **Конечный автомат (FSM):** Для управления процессом заказа была введена `OrderState` (FSM).
- **Пошаговая навигация:** Реализована полная воронка продаж для продукта "Баннеры":
  1. Выбор категории (`/catalog`).
  2. Выбор материала баннера.
  3. Ввод размера (`Ширина x Высота`).
  4. Расчет и отображение итоговой стоимости.

### 2.2. Переработка UI и команд
- **Команды:** Меню команд в Telegram обновлено на `/start`, `/catalog`, `/reset`, `/help`.
- **Клавиатуры:** Созданы новые `inline`-клавиатуры (`get_catalog_keyboard`, `get_banner_materials_keyboard` и др.) и `CatalogCallback` для навигации по воронке.
- **Кнопка "Заказать":** К сообщению с расчетом цены добавлена кнопка "Заказать", ведущая на контакт менеджера (`MANAGER_TELEGRAM_URL`).
*Эта упрощённая реализация для демонстрации MVP. Далее планируется апдейт с сохранением расчётов клиентов и отправкой готовых расчётов менеджеру в виде сформированного заказа. Будет реализована база для сбора статистики и анализа конверсии. На данный момент все действия клиентов в боте уже логируются и собираются в базу для статистики*

### 2.3. Калькулятор цен и визуальное оформление
- **Модуль `pricing.py`:** Создан модуль для инкапсуляции логики расчета цен.
- **Изображения для каталога и расчета:** Добавлены новые конфигурационные переменные (`CATALOG_PHOTO_PATH`, `CALCULATION_PHOTO_PATH`) и соответствующая логика в обработчики для отправки тематических изображений на ключевых шагах воронки.

## III. Ключевые архитектурные изменения

- **Разделение ответственности:** Вся логика форматирования и отправки ответов AI была перенесена из ядра (`app/core/chain.py`) в слой представления (`app/handlers/user_handlers.py`). Теперь RAG-цепочка отвечает только за генерацию контента, а обработчики — за его корректное отображение в Telegram.
- **Инкапсуляция логики:** Созданы и переработаны вспомогательные функции (`_edit_message`, `sanitize_for_telegram_markdown`, `escape_markdown_legacy`), которые инкапсулируют сложную логику и делают код обработчиков чище.

В результате этих изменений проект трансформирован в стабильный, функциональный MVP, готовый к демонстрации и дальнейшему развитию.
