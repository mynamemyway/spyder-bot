# Release v0.2.1: UX & Analytics Improvements

Этот релиз сфокусирован на двух ключевых улучшениях: повышении удобства взаимодействия с "живым калькулятором" и внедрении комплексного сбора статистики по действиям пользователя внутри воронки продаж.

---

## I. Закрепление виджета-калькулятора для улучшения UX

**Проблема:** После нескольких сообщений в чате "живой калькулятор" уходил вверх по истории, и пользователю приходилось его искать, что снижало удобство использования.

**Решение:** Реализован механизм автоматического закрепления виджета, который обеспечивает пользователю постоянный и быстрый доступ к актуальному расчету.

-   **Надежное "тихое" закрепление:**
    -   Создана новая функция-помощник `_pin_message_and_delete_notification`.
    -   Эта функция сначала закрепляет сообщение с флагом `disable_notification=True`, а затем **немедленно удаляет системное уведомление** о закреплении (`message_id + 1`). Этот подход гарантированно избавляет чат от визуального шума, даже если клиент Telegram игнорирует флаг `disable_notification`.
-   **Интеграция:** Вызов новой функции добавлен в обработчики `handle_start` и `handle_catalog` сразу после создания виджета.
-   **Сохранение истории заказов:** В соответствии с финальным решением, логика открепления сообщений **не реализуется**. Это позволяет пользователю иметь список закрепленных сообщений, который служит удобной историей его активных и завершенных расчетов/заказов.

## II. Комплексный сбор статистики по воронке продаж

**Проблема:** Ранее действия пользователя внутри многошаговой воронки продаж (выбор материала, качества печати и т.д.) не фиксировались в базе данных статистики, что делало невозможным детальный анализ конверсии и выявление "узких мест".

**Решение:** Добавлено логирование для всех `callback`-запросов в воронке.

-   **Интеграция в `navigate_catalog`:** В начало обработчика `navigate_catalog`, который является единой точкой входа для всей навигации по воронке, добавлен вызов функции `log_query`.
-   **Информативный формат лога:** Действия пользователя записываются в таблицу `query_stats` в формате `CLICK: {action} (item: {item_id})` (например, `CLICK: select_material (item: frontlit_440)`). Это обеспечивает детальный сбор данных о выборе пользователя на каждом шаге.

Эти изменения делают взаимодействие с ботом более удобным для пользователя и предоставляют ценные данные для анализа эффективности воронки продаж.
